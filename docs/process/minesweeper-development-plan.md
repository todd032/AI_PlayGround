# Mini Minesweeper 개발 계획서

## A. 입력 요약 (기획서 해석)
- 게임 이름: Mini Minesweeper
- 한 줄 소개: 숫자 힌트로 지뢰 위치를 추리해 안전 칸을 모두 여는 퍼즐 게임
- 이번 버전 목표(MVP): 9x9 / 지뢰 10개 / 첫 클릭 안전 / 깃발 / 타이머 / 최고 기록 저장
- 제외 범위(Out of Scope): 난이도 선택, 사용자 맵 설정, 힌트 시스템

## B. 기능 요구사항 (Functional Requirements)
1. 게임 루프
   - 새 게임 시작 → 첫 클릭 시 지뢰 배치 → 승/패 종료 → 재시작
2. 핵심 규칙
   - 지뢰가 아닌 칸 전부 오픈 시 승리
   - 지뢰 클릭 시 즉시 패배
   - 숫자 칸은 인접 지뢰 수를 표시
   - 숫자가 0인 칸은 연쇄 오픈
3. 조작
   - 좌클릭/탭: 칸 오픈
   - 우클릭 또는 롱프레스: 깃발 토글
4. UI/HUD
   - 타이머, 남은 지뢰 추정치, 최고 기록, 상태 메시지, 재시작 버튼
5. 저장
   - `localStorage` 키: `minesweeper-best-time`
   - 승리 시 기록 갱신

## C. 비기능 요구사항 (Non-Functional Requirements)
- 성능 목표: 9x9 보드에서 즉시 반응(지연 체감 없음)
- 호환성: 최신 Chrome/Edge/Safari 모바일/데스크톱
- 반응형: 최소 360px 너비 대응
- 접근성: 버튼 기반 셀, 라이브 상태 메시지 제공

## D. 폴더/파일 설계
- 게임 경로: `games/minesweeper/`
- 파일:
  - `index.html`
  - `styles.css`
  - `game.js`
- 루트 런처(`index.html`)에 게임 카드 추가 필요
- README에 실행/접속 경로 추가

## E. 시스템 설계 (간단)
- 상태 모델: `board`, `firstClickDone`, `gameOver`, `openedCount`, `flaggedCount`, `timer`, `bestTime`
- 엔티티 정의: `cell(row, col, isMine, isOpen, isFlagged, adjacent)`
- 업데이트 루프: 이벤트 기반(클릭/플래그 시 즉시 렌더링), 타이머는 `setInterval`
- 판정 방식:
  - 첫 클릭 셀 + 인접 셀 제외 후 지뢰 무작위 배치
  - 오픈 후 안전 칸 오픈 수가 목표치에 도달하면 승리
- 확장 포인트: 난이도 별 ROW/COL/MINE_TOTAL 상수 분리

## F. 구현 작업 분해 (Task Breakdown)
1) 스캐폴딩
- 산출물: 폴더/HTML/CSS/JS 기본 구조
- 완료 조건: 화면 렌더링 및 보드 생성
- 리스크/대응: 모바일 터치 충돌 → 롱프레스 분기

2) 렌더링/UI
- 산출물: HUD, 상태 문구, 보드 스타일
- 완료 조건: 셀 상태(open/flag/mine) 시각 반영
- 리스크/대응: 숫자 가독성 저하 → 색상 클래스 분리

3) 입력 처리
- 산출물: 좌클릭 오픈, 우클릭/롱프레스 깃발
- 완료 조건: 데스크톱/모바일 모두 동작
- 리스크/대응: 롱프레스 후 click 중복 처리 방지

4) 게임 규칙 구현
- 산출물: 첫 클릭 안전, 지뢰 배치, 인접 수 계산, 연쇄 오픈, 승패 판정
- 완료 조건: 핵심 규칙 모두 정상 동작
- 리스크/대응: 중복 오픈 카운트 오류 → 조건 검사 강화

5) 저장/결과 화면
- 산출물: 최고 기록 저장/로드, 상태 메시지
- 완료 조건: 승리 시 best time 갱신
- 리스크/대응: 저장값 파싱 실패 → 기본값 처리

6) 문서 업데이트
- 산출물: README, 런처 카드, 스크린샷 근거
- 완료 조건: 사용자 진입 경로 명확화
- 리스크/대응: 링크 오타 → 수동 검증

  - 지뢰 클릭 시 패배 처리
  - 안전 칸 전부 오픈 시 승리 처리
  - 우클릭/롱프레스 깃발 토글
  - 최고 기록 저장/표시
